Initial release. Generates iOS Swift client SDKs from an OpenAPI 3.x spec: protocols (one per tag) + `URLSession<Tag>API` default impl, `Codable` structs/enums with format-aware Foundation types, sum-type return for ops with multiple 2xx schemas, and runtime helpers (`APIClient`, `APIError`, `APIInterceptors`, `Auth`, `MultipartFormBody`, `URLEncoding`, `RequestOptions`). Per-call options cover `client` / `baseURL` / `timeout` / `headers` / `requestInterceptors` / `responseValidator` / `responseTransformer`; per-op security from `securitySchemes` is auto-applied via a `client.auth` bag. Every operation also emits a `*WithResponse` overload returning `(T, HTTPURLResponse)` for callers that need response headers. Output ships as raw files (drop into an Xcode target) or as a self-contained SwiftPM library when `package: { name }` is passed. Reuses the same hey-api IR pipeline as `@ahmedrowaihi/openapi-kotlin`. Everything AST-built, no string templating.
